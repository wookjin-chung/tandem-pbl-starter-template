 Tandem PBL Starter Template
 GitHub 기반 판단 기록 학습 공간 (Decision-Oriented Learning Repository)

> 이 레포는 코드 저장소가 아닙니다.
> 이 레포는 판단(Decision)을 기록하고, 실험 과정을 구조화하기 위한 개인 학습 공간입니다.

 1. 이 레포의 목적 (Why)
   대부분의 학습은 “정답 코드”를 따라가며 끝납니다.
   하지만 실제 문제 해결 능력은 코드를 짜는 순간이 아니라, 판단을 내리는 순간에 만들어집니다.

   이 템플릿은 다음을 목표로 합니다:
    ❌ 정답 제공
    ❌ 점수/랭킹
    ❌ 완성 결과 비교
   대신,
    ✅ 가정 → 관측 → 해석 → 다음 실험
    ✅ 판단의 흔적 기록
    ✅ 재시도 설계 능력 강화

2. 핵심 철학 (One Sentence)
   - AI는 정답을 추천하지 않는다.
   - AI는 판단의 흔적을 비추는 거울이다.
   - 최종 결정은 항상 사용자에게 있다.

3. 🚀 시작 방법 (Start Here)
   1️⃣ 이 템플릿으로 내 레포 만들기
     상단의 Use this template 버튼을 눌러
     본인의 학습용 GitHub 레포를 생성하세요.
   
   2️⃣ 첫 번째 Decision Log 작성 (Day 1)
     상단 Issues → New Issue → Decision Log 선택
     아래 4단계를 반드시 채워 기록합니다:
     (1) Assumption (가정)
       “나는 X라고 가정했다”
     (2) Observation (관측)
       “실제로 Y가 관측되었다 (로그/수치/영상/커밋 링크)”
     (3) Interpretation (해석)
       “차이는 Z 때문이라고 본다 (가능한 가설 2~3개)”
     (4) Next Test (다음 실험)
       “다음엔 단 하나의 변수만 바꿔 확인한다”
     📌 정답을 쓰지 마세요. 판단 과정을 쓰세요.
   
     🧪 사용 흐름 예시 (Day 3)
       문제: 로봇이 직진하지 않는다
       ❌ “모터가 불량이다”
       ⭕
          가정: 좌우 PWM 편차가 있을 것이다
          관측: 동일 PWM에서도 좌측이 먼저 회전
          해석: 캘리브레이션 오차 가능성
          다음 실험: 좌측 PWM −5 보정 후 재측정
       이 기록이 학습의 실체입니다.

4. AI 사용 원칙 
   AI는 다음을 절대 하지 않습니다:
   ❌ 코드 자동 생성
   ❌ 정답 추천
   ❌ 점수/평가

   AI가 할 수 있는 것은 오직 아래 3가지입니다:
   [1] Mirror (거울) : “이번에는 조건 점검 → 수정 순서를 밟았습니다.”
   [2] Contrast (비교) : “이전 시도는 코드 수정부터, 이번은 센서 안정화부터 시작했습니다.”
   [3] Question (질문) : “가정을 검증하려면 어떤 변수 1개를 고정해야 할까요?”

5. 성장 지표 (점수 없음)
  이 레포에서는 점수를 매기지 않습니다.
  대신 다음 지표만 스스로 확인합니다:
  - 가정을 명시한 횟수
  - 실험에서 바꾼 변수의 수
  - 재시도 설계의 명확성
  - 근거 링크 첨부율
  - 설명 문장의 명료도

🔒 공개 & 보호 원칙
   공개는 선택 사항
   결과물이 아니라 과정만 기록
   비교/순위/정답 문화 배제

>> 학습자는 평가 대상이 아니라
>> 사고를 실험하는 주체입니다.
